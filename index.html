System Pipeline

HOW IT WORKS

Thirteen sequential stages transform raw financial text into clustered narratives and volatility predictions — all within a single Streamlit interface.

01
Ingestion
Raw Data Loading
Reads all-data.csv (4,846 labelled financial sentences) with automatic encoding detection — handles UTF-8 and latin-1. Auto-detects headerless files and assigns columns accordingly.
pandaslatin-1auto-detect
02
Preprocessing
Text Cleaning & Filtering
Strips whitespace, removes texts shorter than 10 characters, and validates schema. Optional columns (date, sector, return) are detected and used downstream.
str.striplength filterschema check
03
Representation
Semantic Embeddings
Each sentence is encoded into a 384-dimensional dense vector using all-MiniLM-L6-v2. Results are cached with st.cache_data — no recomputation on parameter changes.
sentence-transformersall-MiniLM-L6-v2384-D
04
Reduction
UMAP Dimensionality Reduction
UMAP with cosine metric collapses 384-D to 5-D for clustering and 2-D for visualisation. This critical step — missing from the original code — dramatically improves HDBSCAN cluster quality.
umap-learncosine metricmin_dist=0
05
Clustering
HDBSCAN Narrative Detection
Density-based clustering identifies variable-shape narrative clusters and labels noise points as -1. Returns per-document confidence probabilities. Sidesteps the need to pre-specify cluster count.
hdbscaneom selectionnoise-aware
06
Sentiment
FinBERT Financial Sentiment
ProsusAI/finbert scores each sentence as positive, negative, or neutral — fine-tuned on financial text for far greater accuracy than lexicon methods. TextBlob provided as a fast fallback.
finberttransformerstextblob fallback
07
Themes
TF-IDF Keyword Extraction
Fit globally on all texts, transformed per cluster to extract the top-10 most distinctive bigram keywords per narrative. Correct approach — fitting per-cluster would break term comparability.
tfidfbigrams2000 features
08
Strength
Narrative Strength Scoring
Aggregates document volume, average cluster confidence, and mean sentiment per narrative. Produces a ranked table identifying dominant vs emerging storylines.
volumeconfidencesentiment
09
Graph
Cosine-Weighted Storyline Graph
Builds a NetworkX graph where edges are only drawn between same-narrative documents with cosine similarity > 0.85. Node size encodes sentiment magnitude. Replaces the meaningless complete-subgraph approach.
networkxcosine similaritythreshold=0.85
10
Trend
Temporal Narrative Tracking
When a date column is present, plots daily document volume per narrative over time. Reveals narrative lifecycle — emergence, peak, and decay — critical for investment signal timing.
time seriespivot tableline chart
11
Core Model
Sector Volatility Prediction
Engineers narrative-strength and sentiment features into a daily time series matrix, then trains Ridge Regression, Random Forest, or Gradient Boosting models using TimeSeriesSplit to prevent look-ahead bias.
ridgerandom forestgradient boostingtimeseriessplit
12
Explainability
Feature Importance Analysis
Surfaces which specific narratives (by ID and type — volume vs sentiment) most strongly predict volatility. Provides economic interpretation: which market storylines are driving sector risk.
feature importancesridge coefnarrative attribution
13
Output
Annotated Dataset Export
Exports a downloadable CSV enriched with Narrative ID, cluster confidence, sentiment score, UMAP coordinates — full audit trail for further analysis or academic submission.
csv exportumap coordsfull annotation

Why This Dataset

The Financial PhraseBank dataset contains sentences from financial news annotated by domain experts. The imbalance (neutral-heavy) reflects real market commentary — most statements are factual rather than directional. This makes the clustering task non-trivial and realistic.

Ground-truth sentiment labels enable validation of the FinBERT inference pipeline against expert human annotation, providing a rigorous evaluation baseline rarely available in unsupervised NLP projects.

Technical Stack

ARCHITECTURE

🔤
sentence-transformers
all-MiniLM-L6-v2 encodes each sentence into a 384-D semantic vector space. Cached via st.cache_data for zero recomputation cost.
🗺️
UMAP
Uniform Manifold Approximation preserves local and global structure during reduction from 384-D to 5-D (clustering) and 2-D (visualisation). Cosine metric matches embedding space.
🧠
HDBSCAN
Hierarchical density-based clustering. Naturally handles noise, variable-density clusters, and does not require a pre-specified cluster count. Returns confidence probabilities per point.
💬
FinBERT
ProsusAI/finbert — BERT fine-tuned on financial news corpora. Domain-aware positive/negative/neutral classification. Significantly outperforms lexicon-based TextBlob on financial text.
🕸️
NetworkX
Storyline graph with cosine-similarity weighted edges. Only intra-cluster pairs above a 0.85 threshold are connected — producing a meaningful graph instead of arbitrary complete subgraphs.
📉
scikit-learn
TF-IDF vectorisation, Ridge/RF/GBM models, StandardScaler pipeline, and TimeSeriesSplit cross-validation. Pipeline architecture prevents data leakage through the scaler step.
📊
Streamlit
Interactive 10-step UI with sidebar controls, st.cache_resource for model loading, st.cache_data for compute steps. Dark-themed with IBM Plex Mono typography.
